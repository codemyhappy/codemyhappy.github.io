# 项目分支策略与部署说明

## 分支策略

本项目采用以下分支策略：

### `main` 分支
- 存放Astro项目的源代码
- 这是开发的主要分支
- 所有的功能开发、修复和改进都在此分支上进行

### `gh-pages` 分支
- 用于GitHub Pages部署
- 存放构建后的静态文件
- 由构建脚本自动更新

## 工作流程

1. 在 `main` 分支上进行开发
2. 当准备部署时，运行部署脚本：
   ```bash
   ./deploy.sh
   ```
3. 脚本会自动构建项目并将结果推送到 `gh-pages` 分支
4. GitHub Pages 会自动从 `gh-pages` 分支提供网站服务

## 部署脚本说明

`deploy.sh` 脚本执行以下操作：

1. 提交源码到当前分支，确保源码安全
2. 安装项目依赖
3. 使用 Astro 构建静态文件到 `dist` 目录
4. 切换到 `gh-pages` 分支
5. 将构建的文件复制到根目录
6. 提交更改

## 注意事项

- 不要在 `gh-pages` 分支上直接编辑文件，因为它们会在下次部署时被覆盖
- 所有源代码修改应在 `main` 分支上进行
- 如果首次运行部署脚本，它将自动创建 `gh-pages` 分支
